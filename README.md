## О проекте

**Финалитика** — это мобильное приложение для **ведения личных финансов** и **планирования бюджета**.

Идея проекта — собрать в одном месте все основные пользовательские сценарии, связанные с деньгами:

- учет **доходов** и **расходов**;
- **переводы между счетами**;
- импорт **банковских выписок**;
- анализ финансового поведения;
- планирование целей и проектов;
- контроль регулярных платежей;
- дополнительные инструменты для повседневной финансовой рутины.

Приложение написано на **.NET MAUI** и **C#** и развивается как персональный финансовый помощник с упором на наглядность, автоматизацию и практическую пользу.

---

## Зачем нужна Финалитика

Большинство приложений для учета финансов либо слишком простые, либо не закрывают повседневные сценарии целиком.

**Финалитика** решает эту задачу комплексно:

- помогает быстро увидеть текущее состояние бюджета;
- показывает, **куда уходят деньги**;
- позволяет анализировать скрытые траты;
- помогает планировать будущие расходы и цели;
- автоматизирует часть рутинных операций через импорт выписок;
- объединяет финансы, заметки, платежи, документы и инвестиционные данные в одном приложении.

То есть это не просто “трекер расходов”, а более широкий инструмент для **личного финансового управления**.

## Структура приложения

В приложении предусмотрено **5 основных вкладок** в tab bar.

### 1. Главная
Главная страница — это агрегатор всех основных данных приложения.  
Она показывает ту информацию, которую пользователь должен увидеть в первые секунды после открытия.

Сюда входят:
- сводка по финансам;
- ключевые показатели бюджета;
- быстрый доступ к основным действиям.

Во **Floating Action Button (FAB)** доступны:
- добавление дохода;
- добавление расхода;
- перевод между счетами;
- создание счета;
- создание категории;
- импорт банковской выписки.

---

### 2. Анализ
Раздел анализа — один из центральных сценариев приложения.

#### Индекс погоды личных финансов
Это интерактивный индикатор состояния бюджета на месяц.

Он формируется на основе разницы между доходами и расходами:
- чем сильнее перевес в сторону доходов, тем лучше “погода”;
- шкала может меняться от **финансового шторма** до **солнечной погоды**.

#### Бюджетная аналитика
В разделе представлены:
- графики **денежного потока**;
- структура расходов;
- поиск **скрытых трат**.

Под скрытыми тратами понимаются:
- комиссии;
- подписки;
- регулярные незаметные списания.

#### Анализ поведения
Дополнительно приложение отслеживает:
- **ночные покупки**;
- частые крупные траты по одной категории;
- особенно чувствительные сценарии, например расходы на **шоппинг**.

#### Налоговые вычеты
Есть отдельный блок анализа расходов, которые потенциально можно учитывать для налоговых вычетов.

Сейчас логика строится на поиске по ключевым словам, а в будущем планируется:
- более глубокий анализ доходов и расходов;
- подготовка данных для формирования **декларации 3-НДФЛ**;
- дальнейшая отправка через **личный кабинет ФНС**.

#### Умные рекомендации
Пока это задел на будущее.

Идея в том, чтобы пользователь мог дать **readonly-доступ** через токен AI-системе, после чего приложение смогло бы работать как финансовый ассистент:
- анализировать бюджет;
- подсказывать аномалии;
- давать рекомендации;
- помогать планировать расходы.

FAB в этом разделе аналогичен главной вкладке.

## Импорт банковских выписок

Импорт банковских выписок — одна из наиболее прикладных частей проекта.

### Поддерживаемые банки
На текущий момент поддерживаются:

#### Альфа-Банк
- `XLSX`
- `CSV`

#### Тинькофф Банк
- `CSV`
- `XLSX`
- `OFX`

### Как определяется категория операций
Категоризация доходов и расходов происходит:
- по **MCC-коду**;
- либо по **ключевым словам**.

### Архитектурная основа
В основе реализации импорта лежит паттерн **Factory**.

Это позволяет:
- расширять список банков;
- добавлять новые форматы;
- изолировать логику разбора разных типов выписок.

### Что планируется дальше
В будущем планируется поддержка:
- `PDF`;
- дополнительных банков;
- импорта выписок в формате **1С**.

### Дополнительная логика
Система уже умеет находить **дубли платежей** при импорте и не добавлять их повторно.

## Платежи, заметки и дополнительные инструменты

### Платежи
В приложении можно добавлять платежи с **push-уведомлениями**.

Поддерживаются два типа:
- **разовый платеж** — с конкретной датой и временем;
- **ежемесячный платеж** — с указанием числа месяца и времени уведомления.

### Markdown-заметки
Также в приложении есть общие **markdown-заметки**, которые можно использовать для любых личных нужд:
- планирование;
- списки;
- текстовые заметки;
- черновики финансовых идей.

### Калькулятор зарплаты по графику
Отдельно стоит отметить виджет **«Калькулятор ЗП по графику»**.

Он использует общие данные из профиля, если они указаны:
- должность;
- график;
- ставка или оплата.

Далее пользователь может указать дополнительные параметры:
- расчет зарплаты;
- расчет аванса;
- сменный график;
- тип графика (`2/2`, `3/2` и т.д.).

Результат можно использовать в бюджете, чтобы актуализировать данные для планирования месяца.

## Инвестиции и документы

В приложении реализована **взаимозаменяемая вкладка**:

- **Инвестиции**
- **Документы**

По умолчанию отображается вкладка **Инвестиции**, но в настройках ее можно заменить на **Документы**.

---

### Вкладка “Инвестиции”

Во вкладке инвестиций можно подключить интеграцию с **Tinkoff Invest API** и получать данные по инвестиционным счетам.

Источник:
- [Tinkoff Invest API](https://developer.tbank.ru/invest/intro/intro)

#### Что уже реализовано
- просмотр актуальных данных по портфелю;
- данные по курсам валют;
- отслеживание нужных валют;
- получение ключевой ставки;
- отдельный виджет в стиле **“индекс страха и жадности”**, рассчитываемый на основе индекса **RVI**.

Для валют и ставки используются открытые данные **ЦБ РФ**:
- [Открытые данные ЦБ РФ](https://www.cbr.ru/development/dws/)

#### Что планируется
- ручной ввод инвестиционных данных;
- календарь выплат;
- экспорт и отчеты в PDF.

---

### Вкладка “Документы”

Вкладка документов предназначена для автоматического формирования документов под нужные пользовательские сценарии.

Идея в том, чтобы пользователь:
1. заполнил данные в форме;
2. получил готовый документ;
3. распечатал и подписал его без ручного переписывания.

Поддерживаемые форматы:
- **PDF**
- **XML**

Список документов будет постепенно пополняться.
